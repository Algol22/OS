Практическое задание
1. Вход в систему
На рабочем месте запустите виртуальную машину VmWare и загрузите систему Frenzy v0.3.
Frenzy v0.3 – это дистрибутив системы FreeBSD и имеющий свой специфичный загрузчик.
После полной загрузки вы увидите приглашение командного интерпретатора для ввода логина
и пароля Login:
Чтобы войти в систему наберите root (имя учетной записи суперпользователя – аналог
пользователя administrator в WindowsNT). С учетом особенностей системы Frenzy ввод
пароля не требуется.
Многие Unix системы позволяют пролистывать содержимое консоли вверх(когда нужная
информация «убежала» за границы экрана). Для этого нужно нажать клавишу ScrollLock и
клавишами «», «» пролистать экран. Для продолжения работы еще раз нажать ScrollLock.
2. Просмотр и перемещение по директориям.
Сразу после входа в систему под пользователем root вы окажетесь в каталоге /root – это
домашний каталог суперпользователя (это единственное исключение: все вновь создаваемые
пользователи и их начальные каталоги по умолчанию помещаются в каталог /home).
Проверьте, верно ли это утверждение. Для этого наберите команду pwd.


/////

pwd


Далее просмотрите файлы, находящиеся в этой директории(команда ls).

////
ls


Перейдите в каталог /root (либо тот, который указал преподаватель). Просмотрите его


////
sudo -i
cd /root
ls

содержимое в кратком и полном форматах (команда ls).
3. Создание каталогов и файлов.
В каталоге /root (либо том, который указал преподаватель) попробуйте создать свой
домашний каталог вида /root/<свое имя>. (Например /root/name). Для создания
каталога воспользуйтесь командой mkdir.

Скопируйте файл /etc/rc.conf в ваш домашний каталог. Воспользуйтесь командой cp.


////
cp etc/apg.conf /root/andrey

Просмотрите его содержимое с помощью утилиты просмотра текстовых файлов less. Ее
параметры и ключи самостоятельно найдите на страницах справочного руководства. Для этого
необходимо набрать man less. Для выхода из утилиты просмотра необходимо нажать

/////
man less
клавишу «q».
Переименуйте файл /root/name/rc.conf под именем /root/name/name1.conf.

//////
mv root/andrey/apg.conf /root/andrey/andrey1.conf



Сделайте 3 копии этого файла с именами name2.conf name3.conf name4.conf.



////
cat /root/andrey/andrey1.conf | tee andrey2.conf andrey3.conf andrey4.conf




Скопируйте каталог /etc с вложенными подкаталогами и файлами в ваш домашний каталог.

////
mkdir /root/andrey/yyy/


Для этого необходимо вести команду cp с ключом рекурсивного копирования.
Самостоятельно найдите этот ключ на страницах руководства. Теперь просмотрите
содержимое скопированного каталога в полном формате.
Скопируйте каталог /root/name/etc в каталог /root/name/eee


/////
cp -R ../etc /root/andrey/yyy



Создайте еще одну пустую директорию с именем /root/name/conf/. Переместите туда 2

////
mkdir /root/andrey/conf

файла name3.conf и name4.conf.

//////
mv andrey3.conf andrey4.conf /root/andrey/conf/

4. Удаление файлов и каталогов.
Удалите файл name3.conf из директории /root/name/conf/.

/////
rm /root/andrey/conf/andrey3.conf


Перейдите в каталог /home/name/eee/. 
///
cd /root/andrey/eee



Удалите рекурсивно все файлы и подкаталоги этого
каталога. Проверьте, получилось ли у вас это.

////
rm -R /root/andrey/eee
