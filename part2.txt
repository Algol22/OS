Практическое задание
1. Войти в систему с собственной учетной записью.
2. Создать в домашнем каталоге 3 файла произвольного содержания (имена файлов - a1,
a2, a3).


touch a1 a2 a3



3. Получить развернутый список файлов домашнего каталога и сохранить его в файле



touch listing1
ls -la >> listing1
		
		
4. Просмотреть файл listing1, обратив внимание на поля прав доступа, владельца и
группы.

cat listing1 | grep listing1




5. В tty2 открыть сеанс root с помощью sudo -i.

su


6. Перейти в каталог /home/usertk.


cd /home/user

7. Изменить права доступа к файлам a1 и a2 следующим образом:
 a1: запретить запись для владельца и группы
chmod gu-w a1

 a2: разрешить запись для всех
chmod a+w a2

8. Переключиться в tty1 и изменить содержимое файлов a1 и a2. Сохранить изменения.

exit
echo text for a1 > a1
PERMISSION DENIED
	echo text for a2 > a2
	cat a2
		
	
9. Перейти в tty2 и изменить владельца файлов a1 и a2 на root

chown root a1 a2


10. Перейти в tty1

exit


11. Из tty1 попробовать изменить файл a2

echo changes >> a2

12. Получить развернутый список файлов каталога /home/usertk и дописать его в файл
listing1.


ls -la >> listing1

13. Просмотреть файл listing1 и сравнить предыдущие и текущие права доступа,
владельца и группы.


cat listing 1 | grep 'a1\|a2'


14. Перейти в tty2

su

15. Установите системный признак неудаляемости для файла a3.

chattr +i a3
16. Попытайтесь удалить файл a3. Просмотрите права на удален

rm a3
lsattr a3
